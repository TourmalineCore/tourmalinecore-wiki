# tourmalinecore-wiki
This repository is the knowledge base of the Tourmaline Core company. The main goal is to store the most important information, guidelines, instructions, useful links, etc. regarding basic technologies and approaches used within the company.

It should help to gather and keep the information in one place in order to increase quality of knowledge transfer between the teams, to decrease the loss of knowledge, and make the new team members onboarding process easier and more efective.  

This knowledge base doesn't include information regarding specific projects. It aims to store the knowledge only about the specific technologies that could be reused in several project. All information regarding projects domain and its sole non-reusable nuances must be stored in projects' repositories.

Anyone from Tourmaline Core team could contribute to this repository. Pull requests must be approved by Aleksandr S. before pushing to the master branch.

# File structure
An assumed list of folders:

/frontend
/backend  
/design  
/testing
/embeded  
/ml  
/devops  
/management
/languages
/shared-technologies

Root folders in the file structure should contain the information regarding technologies and approaches of specific development direction. (e.g. /frontend folder contains guides and articles only regarding frontend development). 
Please don't save any files without folder.

Folder /languages should contain specific information about languages, which could be used independently direction of development (e.g. C++, Python). 

Folder /shared-technologies should contain information about specific technologies or approaches which could be used independently direction of development (e.g., local env, docker etc.).

If you are going to share some materials from the internet, please create file useful-links.md in folder of direction, and put the link there.

If you are going to share important literature (books, articles), please create file useful-books.md in folder of direction, and put the information there.
